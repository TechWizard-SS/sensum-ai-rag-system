# Sensum AI RAG System

**Sensum AI RAG System** — это учебный MVP-проект, реализующий систему генерации ответов с дополнением извлечёнными данными (RAG) на основе собственной базы знаний. Проект нацелен на создание полностью изолированного от интернета решения для безопасного хранения и использования интеллектуальной собственности (документов, статей, презентаций) внутри компании. Несмотря на использование облачных сервисов (OpenRouter, Supabase) на этапе разработки, архитектура позволяет легко перейти на локальную LLM и самохостинг базы данных.

> **Важно:** Проект находится в стадии MVP, содержит ряд упрощений и «костылей» (например, самоинжекция сервисов), которые будут исправлены в следующих версиях. Тем не менее, он полностью работоспособен и может быть использован как основа для корпоративной базы знаний малого и среднего бизнеса.

---

## 🚀 Возможности

- **Загрузка документов** — поддержка форматов **PDF**, **Word (.docx)** , **PowerPoint (.pptx)** , а также парсинг **web-статей** по URL.
- **Векторное хранилище** — на основе **pgvector** (через Supabase) для семантического поиска релевантных фрагментов.
- **Три режима чата**:
  - `FAST` — обычный диалог с ИИ без использования базы знаний (только контекст чата, ограниченный **7 последними сообщениями**).
  - `DEEP` — семантический поиск по базе знаний, вывод **15 наиболее релевантных фрагментов** в виде карточек с возможностью просмотра исходной страницы (с поддержкой рисунков) и выбора конкретного чанка для отправки в ИИ.
  - `AI` — автоматический выбор **топ-5 релевантных чанков** (только текст) и генерация ответа с их учётом.
- **Режим Audit** — при включении загруженные в чат документы **не сохраняются** в базу знаний, а остаются только в контексте (полезно для работы с черновиками).
- **Гибкость провайдеров** — используется **OpenRouter** (поддерживаются различные модели), но возможна замена на локальную LLM (например, через Ollama).
- **Два способа запуска** — через Python-лаунчер или напрямую через Gradle (фронтенд статически раздаётся бэкендом).

## 📦 Установка и запуск

### Требования
- Java 21
- Node.js 18
- PostgreSQL с pgvector (или аккаунт Supabase)
- Gradle
- Python

### 1. Клонирование репозитория
```bash
git clone https://github.com/TechWizard-SS/sensum-ai-rag-system.git

cd sensum-ai-rag-system
```

2. Настройка переменных окружения
Задайте переменные в системе

# OpenRouter
OPENROUTER_API_KEY=your_openrouter_api_key
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1

# Supabase
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key
SUPABASE_STORAGE_BUCKET=documents  # имя bucket для хранения файлов

# База данных (используется из Supabase, но можно указать отдельно)
SPRING_DATASOURCE_URL=jdbc:postgresql://<host>:<port>/<db>
SPRING_DATASOURCE_USERNAME=postgres
SPRING_DATASOURCE_PASSWORD=your_password

# Настройки приложения (опционально)
RAG_CONTEXT_LIMIT=7          # количество сообщений в контексте FAST режима
RAG_TOP_K=15                 # количество возвращаемых чанков в DEEP
RAG_AI_TOP_K=5               # количество чанков для AI режима
Примечание: Для разработки используется бесплатный тариф Supabase (лимиты: Storage — 1 ГБ, база данных — 500 МБ). При переходе на прод рекомендуется развернуть собственную PostgreSQL с pgvector и локальное S3-хранилище (MinIO).

3. Сборка и запуск
Способ 1: Через Python-лаунчер
```bash
# Установите зависимости Python (если ещё нет)
pip install -r requirements.txt  # если есть, иначе просто запускаем

# Запуск лаунчера (он соберёт бэк и фронт, затем запустит Spring Boot)
python launcher.py
Способ 2: Через Gradle (ручной запуск)
bash
# Сборка бэкенда
./gradlew clean build

# Сборка фронтенда (статический вывод в src/main/resources/static)
cd frontend
npm install
npm run build
cd ..

# Запуск Spring Boot приложения
./gradlew bootRun
```

🔧 Использование
Основные сценарии
Загрузка документов

Перейдите в раздел «База знаний».

Загрузите файл (PDF, DOCX, PPTX) или вставьте URL веб-статьи.

Документ будет обработан, разбит на чанки, векторизован и сохранён в pgvector.

Режимы чата

FAST — используйте для быстрых вопросов без обращения к базе знаний.

DEEP — задайте вопрос, получите 15 релевантных фрагментов в виде карточек. Нажмите на карточку, чтобы увидеть исходную страницу (с рисунками). Выберите один или несколько чанков и отправьте в ИИ для детального ответа.

AI — всё автоматически: поиск топ-5 чанков, генерация ответа с их учётом.

Режим Audit

Включите тумблер «Audit» перед загрузкой документа в чат. Документ будет использован только в текущем диалоге и не попадёт в общую базу знаний.

⚠️ Известные ограничения и «костыли»
Контекст чата — жёстко ограничен 7 последними сообщениями (для экономии токенов). На проде требуется реализовать сжатие истории.

Самоинжекция сервисов — в некоторых местах используется @Autowired с self-injection для разрешения циклических зависимостей. Это будет исправлено рефакторингом.

Отсутствует гибридный поиск — сейчас только семантический (векторный). Планируется добавить полнотекстовый поиск (BM25) и ранжирование.

Переключение моделей в настройках — функционал нестабилен, требует доработки (пингование доступности, корректная смена провайдера).

Безопасность — в текущей версии отсутствует авторизация пользователей (только один общий доступ). Для продакшена необходима реализация аутентификации.

Добавление пользовательской аутентификации и ролевой модели.

Рефакторинг архитектуры (устранение самоинжекций, улучшение тестируемости).

Пингование внешних сервисов и автоматическое переключение на резервные.

Расширение поддерживаемых форматов (Excel, изображения с OCR).
